# CLAUDE.md

## Project Overview

laudec — "See everything Claude Code does. One binary. One command."

A Rust binary that wraps Claude Code, intercepting all API traffic and OTEL telemetry into a local SQLite database, with a built-in web dashboard for inspection.

## Architecture

```
Claude Code  ──▶  laudec proxy (Axum)  ──▶  Anthropic API
     │                    │
     │                    ▼
     │              SQLite (api_calls)
     │
     └──▶  laudec OTEL collector (tonic/gRPC)
                          │
                          ▼
                    SQLite (otel_events, otel_metrics)
                          │
                          ▼
                    Dashboard (Svelte SPA served by Axum)
```

### Key ID relationship
- `run_id` (UUID) — generated by laudec per invocation, tags `api_calls` rows
- `cc_session_id` — Claude Code's native session ID from OTEL events
- `session_id_map` table links them (written at OTEL ingest time + post-session fallback)

## Source Layout

| File | Role |
|------|------|
| `src/main.rs` | CLI (clap), orchestrates proxy/collector/dashboard/launcher |
| `src/proxy.rs` | HTTP proxy — intercepts API calls, logs to DB |
| `src/collector.rs` | gRPC OTEL collector — receives logs/metrics from CC |
| `src/dashboard.rs` | Axum API routes + serves embedded Svelte SPA |
| `src/db.rs` | SQLite schema, all DB methods (spawn_blocking + conn.lock()) |
| `src/session.rs` | Post-session bookkeeping (stats, git diff, summary) |
| `src/config.rs` | Config loading (laudec.toml → ~/.config/laudec/ → defaults) |
| `src/launcher.rs` | Spawns `claude` with env vars pointing to proxy/collector |
| `src/settings.rs` | Temporarily patches CC settings, restores on drop |
| `dashboard/` | Svelte 5 SPA (vite build → dashboard/dist/, embedded via rust-embed) |

## Build

```bash
cd dashboard && npm install && npm run build  # frontend first
cargo build --release                          # embeds dashboard/dist/
```

Both steps required — `build.rs` creates a placeholder if dashboard isn't built.

## Database

SQLite at `~/.local/share/laudec/laudec.db` (override with `LAUDEC_DATA_DIR`).

Tables: `api_calls`, `otel_events`, `otel_metrics`, `sessions`, `session_id_map`.
Schema uses `CREATE TABLE IF NOT EXISTS` — no migration system, columns added via `ALTER TABLE` ignoring errors.

## Config

Loaded from `laudec.toml` in project dir, falls back to `~/.config/laudec/config.toml`, then defaults. Generate starter with `laudec init`.

Default ports: proxy 18080, collector 14317, dashboard 18384.

## Testing

```bash
./test_integration.sh   # requires Claude Code installed + API key
```

Runs a real CC session with test prompt, validates DB contents and dashboard API.
